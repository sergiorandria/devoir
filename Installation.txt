# Installation de Mysql-8.3.0 

Veuillez vous premunir de la derniere version de Mysql du site officiel de Mysql. Les pre-requis pour l'installation sont:
	.cmake(apt-get install cmake)
	.make(apt-get install make)
	.les librairies SSL(apt-get install openssl) 
	.Les librairies Boost(apt-get install boost)
	.ncurses(apt-get install libncurses-dev)
	.Perl(apt-get install perl)

Pour la pre-configuration, creer un nouveau groupe mysql:
	# groupadd mysql
Ensuite, ajouter l' utilisateur mysql a ce nouveau groupe:
	# useradd -r -g mysql -s /bin/false mysql 
On peut maintenant proceder a la suite de l' installation. 
Decompresser l'archive mysql-8.3.0 avec la commande tar:
	# tar -xvzf mysql-8.3.0
	
A l' issue de cette commande, un nouveau repertoire va etre cree. Acceder a ce nouveau repertoire avec la commande :
	# cd mysql-8.3.0

Lister le contenu du repertoire pour voir les fichiers de configuration:
	# ls 
	man/ 
	build/
	docs/
	ext/
	libs/
	include/
	...
	...
	
Il se peut que le repertoire build est deja cree. Sinon, creer-le avec:
	# mkdir build
Puis, acceder a ce nouveau repertoire:
	# cd build
Pour creer le makefile, executer la commande;
	# cmake ..
Ensuite, executer la commande make et make install pour la compilation:
	# make && make install
	
S'il y a des erreurs de syntaxes, ajouter l' option -i a make et make install:
	# make -i && make -i install
Si l'erreur persiste, essayer de corriger les erreurs de syntaxe ou telecharger une autre source ou une autre version de Mysql.

Pour la post-installation, acceder au repertoire mysql dans /usr/local:
	# cd /usr/local/mysql
Creer un nouveau repertoire mysql-files;
	# mkdir mysql-files
Changer le proprietaire du nouveau repertoire avec chown:
	# chown mysql:mysql mysql-files
Ensuite, les commandes suivantes  pour initialiser le serveur Mysql;
	# mysqld --initialize --user=mysql
	# mysql_ssl_rsa_setup
	# mysqld_safe --user=mysql &
	
Pour que le service soit reconnue par systemctl ou par service, mysql.server dans /etc/init.d/mysql.server:
	# cp support-files/mysql.server /etc/init.d/mysql.server


# Installation d'apache2 


Pour l' installation d'apache2 sur linux, veuillez vous premunir d'une code source d'apache2 que vous pouvez télécharger sur le site officiel d'apache2 ou d'une branche sur github. 
Nous allons ici proceder a une installation locale, c'est-a-dire dans un repertoire que l'on va creer dans /usr/local(Par defaut pour certain distribution de linux, l' installation d'un paquet sera dans ce repertoire). Si des droits de superutilisateur sont requises, n'hesitez pas a ajouter 'sudo' au debut de chaque ligne de commande

Avant toutes choses, executer la commande:
	# apt-get update

Ensuite, decompresser le fichier .tar.gz avec la commande :
	# tar -zxvf httpd-'version'.tar.gz (l'option -v est optionnel si vous voulez desactiver le verbose)

A l'issue de cette commande, un nouveau repertoire sera cree. Acceder a ce nouveau repertoire :
	# cd httpd-2.4.59
Puis:
	# ls 
		.
		..
		modules/
		os/
		server/
		srclib/
		support/
		test/
		docs/
		include/
		change-entries/
		...
		...	 
		configure
		...
		...
Le resultat peut differer d'une source a l'autre. Le script 'configure' est le script d'installation. Avant de continuer, creer d'abord le repertoire apache2.4.59 dans /usr/local:
	# mkdir /usr/local/apache2.4.59
	
Pour installer apache2 dans le repertoire nouvellement cree:
	# buildconf
	# ./configure --prefix=/usr/local/apache2.4.59 --enable-shared=max
	
Si il y a une erreur lors du processus d'installation:
	"checking for APR... no
	 configure: error: APR not found.  Please read the documentation.", 
Installer les paquets suivants:
	 build-essential libssl-dev libexpat-dev libpcre3-dev libapr1-dev libaprutil1-dev 
avec apt-get:
	# apt-get install build-essential libssl-dev libexpat-dev libpcre3-dev libapr1-dev libaprutil1-dev 
Puis refaire la commande:
	# ./configure --prefix=/usr/local/apache2.4.59 --enable-shared=max

Une fois terminee, executer les commandes make et make install pour installer le serveur apache sur votre systeme:
	# make && make install
Si des erreurs apparaissent, ajouter l'option -i avec make et make install:
	# make -i && make -i install
Si l'erreur persiste, il est preferable de telecharger une autre source ou une autre version d'apache. Pour les plus avances, modifier directement les erreurs de syntaxes dans les fichiers de configuration d'apache2. 
	
Pour lancer le service apache, executer la commande apachectl du repertoire bin/ du repertoire d'installation:
	# /usr/local/apache2.4.59/bin/apachectl start
	
Pour voir le status du service:
	# /usr/local/apache2.4.59/bin/apachectl status 
ou :
	# ps -ef | grep apache2
	

# Installation de php



Veuillez vous munir d'une source fiable de php.
Decompresser l'archive php-8.3.6.tar.gz avec la commande:
	# tar -xvfz php-8.3.6.tar.gz 
A l'issue de cette commande, une nouvelle repertoire va etre cree. Acceder a ce repertoire:
	# cd php-8.3.6
Lister le contenu  du repertoire avec la commande ls, pour voir les fichiers de configurations:
	# ls
	include/
	libs/
	main/
	build/
	...
	...
	buildconf
	configure
	...
	...

Les fichiers buildconf et configure sont les fichiers de configurations
Pour le pre-installation, installer d'abord les dependances:
	build-essential autoconf re2c bison libsalite3-dev libpq-dev libonig-dev libfcgi-dev libfcgi0ldbl libjpeg-dev libpng-dev libssl-dev libxml2-dev libcurl4-openssl-dev libxpm-dev 
libgd-dev libmysqlclient-dev libfreetype6-dev libxslt1-dev libpspell-dev libzip-dev libgccjit-10-dev
avec apt-get:
	# apt-get install build-essential autoconf re2c bison libsalite3-dev libpq-dev libonig-dev libfcgi-dev libfcgi0ldbl libjpeg-dev libpng-dev libssl-dev libxml2-dev libcurl4-openssl-dev libxpm-dev 
libgd-dev libmysqlclient-dev libfreetype6-dev libxslt1-dev libpspell-dev libzip-dev libgccjit-10-dev 

Pour generer la configuration:
	# ./buildconf

Ensuite, configurer la compilation avec:
	# ./configure

Si vous voulez l'installer dans un repertoire specifique, ajouter l'option --prefix a ./configure:
	# ./configure --prefix='chemin d'acces'

Ensuite, compiler le programme avec make et make install:
	# make && make install

S'il y a des erreurs de syntaxes dans les fichiers de php, ajouter l'option -i a make et make install:
	# make -i && make -i install
